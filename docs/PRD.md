## 一、产品背景与目标

### 1.1 业务背景

在东莞市数标园运营的独立软件开发者/一人软件公司面临以下痛点：

1. **客户管理混乱**: 潜在客户信息散落在微信、Excel、笔记本中，跟进记录无法追溯
2. **项目进度失控**: 多项目并行时，任务优先级不清，交付延期风险高
3. **财务记录缺失**: 合同金额、收款进度、支出记录不透明，无法准确核算利润
4. **缺乏数据支撑**: 经营决策凭感觉，无法量化分析客户转化率、项目利润率
5. **工具碎片化**: 使用多个工具（Excel、微信、记事本）管理业务，效率低下

### 1.2 产品目标

| 目标 | 量化指标 |
|------|----------|
| 客户管理效率提升 | 客户信息录入时间 < 30秒，跟进记录完整率 > 90% |
| 项目交付准时率 | 项目延期率降低 40% |
| 财务透明度 | 收支记录准确率 100%，月度报表生成时间 < 1分钟 |
| 经营决策支持 | 提供 5+ 核心业务指标看板 |
| 工具整合 | 替代 3+ 个独立工具，统一到一个平台 |

### 1.3 产品愿景

成为一人软件公司的"操作系统"，让开发者专注于写代码，而不是管理业务。

## 二、用户画像

### 2.1 核心用户：独立软件开发者

**基本信息**
- 年龄：25-40岁
- 地点：东莞市数标园及周边
- 经验：3-10年软件开发经验
- 团队规模：1人（偶尔外包协作）

**业务特征**
- 年接项目：5-15个
- 项目类型：企业官网、小程序、管理系统、API开发
- 项目周期：2周 - 3个月
- 客单价：5,000 - 100,000元

**核心诉求**
1. 快速记录客户信息和需求
2. 清晰掌握每个项目的进度和待办
3. 清楚知道应收/已收/待收金额
4. 一眼看到本月/本季度经营状况

**使用场景**
- 场景1：接到新咨询电话 → 30秒内录入客户信息 → 设置跟进提醒
- 场景2：项目启动 → 创建项目 → 分解任务 → 设置里程碑 → 每日更新进度
- 场景3：收到客户付款 → 记录收入 → 关联合同 → 自动更新项目状态
- 场景4：月末复盘 → 查看仪表盘 → 分析本月营收、客户转化、项目健康度

### 3.2 详细功能说明

#### 模块一：认证系统

**功能描述**: 管理员登录认证，保障系统安全

**用户故事**:
- 作为用户，我希望通过用户名密码登录系统
- 作为用户，我希望登录后保持会话，无需频繁重新登录
- 作为用户，我希望可以修改密码

**功能清单**:

| 功能 | 详细说明 | 验收标准 |
|------|----------|----------|
| 登录 | 用户名+密码登录，JWT Token认证 | 密码错误提示明确，登录成功跳转首页 |
| Token刷新 | 访问令牌过期后自动刷新 | 无感刷新，用户体验流畅 |
| 登出 | 清除本地Token | 登出后无法访问受保护页面 |
| 初始管理员 | 系统首次启动创建默认管理员 | 用户名: admin, 密码: admin123 |

**业务规则**:
- 密码长度 6-20 位，需包含字母和数字
- 登录失败 5 次锁定账户 30 分钟
- Access Token 有效期 30 分钟
- Refresh Token 有效期 7 天

| 营收趋势图 | 月度营收折线图 | 最近12个月数据 |
| 客户转化漏斗 | 潜在→跟进→成交 | 显示各阶段数量和转化率 |
| 项目状态分布 | 各状态项目数量饼图 | 实时更新 |
| 待办事项 | 显示待处理任务/即将到期合同 | 按优先级排序 |
| 快捷操作 | 快速新建客户/项目/合同 | 一键跳转 |

**核心指标定义**:

| 指标 | 计算方式 | 更新频率 |
|------|----------|----------|
| 本月收入 | 本月确认收入总额 | 实时 |
| 本月支出 | 本月确认支出总额 | 实时 |
| 本月利润 | 本月收入 - 本月支出 | 实时 |
| 进行中项目 | 状态非交付/暂停的项目数 | 实时 |
| 客户转化率 | 成交客户数 / 总客户数 × 100% | 实时 |
| 项目延期率 | 延期项目数 / 总项目数 × 100% | 实时 |
| 应收账款 | 已生效合同金额 - 已确认收入 | 实时 |
