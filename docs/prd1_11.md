# 数标云管 v1.11 — Claude Code 执行指令（数据与模型交付模块，修正版 R2）

## 项目目标

在已完成的 v1.0 ~ v1.10 代码库基础上，新增数据标注与模型开发交付台账能力。

本版本不做：在线标注平台、模型训练编排、自动评测平台、多人协作、GPU调度、文件哈希自动计算。

---

## 零、关键语义边界定义（实施前必读，禁止跳过）

### 0.1 本版本的定位

只做：记录、关联、追踪、验收。不做：执行、调度、计算、平台化。

### 0.2 核心交付链路（必须在数据库层锁死）

```
Project → Dataset → DatasetVersion
                         ↓ (link, 自动 in_use)
               TrainingExperiment
                         ↓ (产出)
                   ModelVersion
                         ↓ (打包)
                 DeliveryPackage ──delivery_package_id──→ Acceptance（复用 acceptances 表）
```

验收记录通过 delivery_package_id 外键直接关联交付包，链路完整闭合，禁止断链。

### 0.3 与 v1.5 现有表的关系（禁止重复建模）

```
acceptances 表：新增 delivery_package_id（外键，必填）+ 新增 acceptance_type 枚举值：dataset/model
deliverables 表：新增 delivery_package_id（外键，可选）
requirements 表：新增 annotation_task_id（外键，可选）+ 新增 requirement_type 枚举值：annotation_spec
requirement_changes 表：直接复用，无需修改
```

### 0.4 版本冻结规则（禁止违反）

```python
# DatasetVersion 冻结：status in (ready, in_use, archived)
#   不可改：version_no / dataset_id / sample_count / file_path / data_source / label_schema_version
#   可修改：notes / change_summary
#   错误码：DATASET_VERSION_FROZEN

# ModelVersion 冻结：status in (ready, delivered, deprecated)
#   不可改：version_no / experiment_id / name / file_path / metrics
#   可修改：notes
#   错误码：MODEL_VERSION_FROZEN

# TrainingExperiment 冻结：一旦关联了任意 DatasetVersion
#   不可改：project_id / framework / hyperparameters
#   可修改：name / description / metrics / notes / started_at / finished_at
#   错误码：EXPERIMENT_FROZEN
```

### 0.5 数据集版本元数据规范（R2新增）

```python
# data_source: TEXT NULL — 数据来源描述，人工填写
# label_schema_version: VARCHAR(50) NULL — 对应标注规范版本号
# change_summary: TEXT NULL — 与上一版本的变更说明，任何冻结状态均可修改
```

### 0.6 标注任务与需求规范的语义边界

```
requirements 表（annotation_spec 类型）：负责"规范是什么"
annotation_tasks 表：负责"这批任务怎么跑、如何质检、是否返工"
两者不互相替代，禁止在 annotation_tasks 表存储规范内容
```

### 0.7 数据集版本状态定义

```
draft   = 可修改核心字段
ready   = 核心字段冻结，可用于训练
in_use  = 被训练实验引用（系统自动设置，接口不可直接传入），核心字段冻结，不可删除
archived = 核心字段冻结，不再使用
```

#### 核心规则

- 数据集必须关联项目（project_id 必填）

- `in_use` 版本不可删除（VERSION_IN_USE_CANNOT_DELETE）

```http
POST/GET/PUT/DELETE /api/v1/delivery-packages

```

#### 核心规则

- 流转到 ready 前必须至少关联一个模型版本或数据集版本
- 状态流转：draft→ready→delivered→accepted
- `PATCH /deliver`（原子事务）：delivery_packages.status=delivered + delivered_at + 批量更新关联 model_versions.status=delivered
- `POST /acceptance`：
  - **delivery_package_id 必填**（锁定到当前交付包，见零章 0.2），写入 acceptances 表
  - acceptance_type 根据包内容自动判断（model 或 dataset）
  - result=passed 时原子事务更新交付包 status=accepted
  - 一个交付包只能有一条验收记录
- accepted 不允许删除（ACCEPTED_PACKAGE_CANNOT_DELETE）

delivery_packages → acceptances(delivery_package_id) → status/notes
