# v2.0 AI Agent 前端手动测试计划

## 测试准备
1. 启动服务：`start.bat`
2. 登录：http://localhost:5173 （admin / admin123）
3. 确认 Ollama 运行中：`curl http://localhost:11434/api/tags`

---

## 一、经营决策 Agent 测试

### 1.1 规则模式测试
1. 进入「经营决策」页面
2. 确认 Provider 标签显示「规则模式」
3. 点击「运行经营分析」
4. 观察：建议列表是否按优先级分组显示
5. 检查：建议卡片是否显示「来源：规则」

### 1.2 LLM 增强测试（gemma4:e2b）
1. 开启「启用 AI 增强」开关
2. 再次点击「运行经营分析」
3. 等待 3-5 分钟（关闭推理模式）
4. 观察：建议内容是否变得更人性化
5. 检查：来源标记是否变为「本地AI」

### 1.3 建议确认闭环
1. 点击某条建议的「接受」按钮
2. 在确认弹窗中：
   - 选择 reason_code
   - 勾选「记入下次分析参考」
   - 点击确认
3. 检查：该建议是否从列表消失
4. 验证：是否创建了对应的待办或提醒

### 1.4 拒绝测试
1. 点击某条建议的「拒绝」按钮
2. 选择 reason_code
3. 点击确认
4. 检查：状态是否变为「已拒绝」

---

## 二、项目管理 Agent 测试

### 2.1 项目智能分析
1. 进入一个「进行中」状态的项目
2. 点击「项目智能分析」按钮
3. 观察：是否返回该项目相关的建议
4. 检查：是否只显示 task_delay 和 change_impact 类型

---

## 三、Agent 运行日志测试

1. 进入「Agent 运行日志」页面
2. 检查列表是否显示：
   - 运行时间
   - Agent 类型
   - 状态
   - Provider 模式
   - LLM 增强状态
3. 点击某条记录展开详情
4. 验证：llm_provider 和 llm_model 字段是否可见

---

## 四、三档 Provider 切换测试

### 4.1 none 模式
1. 修改 .env：`LLM_PROVIDER=none`
2. 重启后端
3. 运行 Agent，观察：是否降级为规则输出

### 4.2 local 模式
1. 修改 .env：`LLM_PROVIDER=local`
2. 确认 Ollama 运行中
3. 运行 Agent，观察：是否调用本地模型

### 4.3 api 模式
1. 修改 .env：`LLM_PROVIDER=api`
2. 配置 LLM_API_KEY
3. 运行 Agent，观察：是否调用外部 API

---

## 五、7 步完整闭环演示

| 步骤 | 操作 | 预期结果 |
|------|------|----------|
| 1 | 打开经营决策页 | Provider 标签显示「规则模式」 |
| 2 | 点击「运行经营分析」 | 展示建议列表，来源「规则」 |
| 3 | 接受一条逾期回款建议 | 创建待办/提醒，勾选「记入下次分析」 |
| 4 | 开启 AI 增强，再次运行 | 建议语言化，来源「本地AI」 |
| 5 | 查看 Agent 运行日志 | 两次运行记录对比 |
| 6 | 验证第二次 context | 包含「用户通常重视逾期回款提醒」 |
| 7 | 验证待办列表 | 催款待办已创建 |

## 六、异常场景测试

| 场景 | 操作 | 预期结果 |
|------|------|----------|
| Ollama 未启动 | LLM_PROVIDER=local，Ollama 关闭 | 降级为规则模式，不报错 |
