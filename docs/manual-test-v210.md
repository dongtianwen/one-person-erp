# v2.1 手动测试计划

## 前置准备

1. **启动后端**：`cd backend && python -m uvicorn app.main:app --reload --port 8000`
2. **启动前端**：`cd frontend && npm run dev`
3. **确认 .env 配置**：
   - `LLM_PROVIDER=api`（测试自由问答需要）
   - `LLM_API_BASE`、`LLM_API_KEY`、`LLM_API_MODEL` 已填写
   - 如无 API Key，可设 `LLM_PROVIDER=none` 测试降级场景
4. **准备测试数据**：
   - 至少 1 个客户 + 1 个 status=completed 的项目（测试报告生成）
   - 至少 1 个交付包（测试质检）
   - 至少几条财务记录（测试问答数据上下文）

---

## 场景一：经营数据自由问答

| 步骤 | 操作 | 预期结果 |
|------|------|---------|
| 1.1 | 侧边栏点击「经营助手」或访问 `/assistant/qa` | 进入问答页面 |
| 1.2 | 页面顶部显示 Provider 状态标签 | `LLM_PROVIDER=api` 时显示绿色标签 + 模型名；`none` 时显示蓝色「API 未配置」 |
| 1.3 | **Provider=api**：输入「最近三个月收入如何？」点击发送 | 显示用户气泡 + 助手回答气泡，回答基于实际数据 |
| 1.4 | 继续追问「哪些项目还有待收款？」 | 多轮对话，助手结合上下文回答 |
| 1.5 | 刷新页面 | 对话历史清空，页面恢复空白 |
| 1.6 | **Provider=none**：访问 `/assistant/qa` | 显示说明页「此功能需接入外部模型」，无输入框 |
| 1.7 | 点击「Agent 设置」链接 | 跳转到 Agent 设置页 |

## 场景二：项目复盘报告

| 步骤 | 操作 | 预期结果 |
|------|------|---------|
| 2.1 | 进入一个 **status=completed** 的项目详情 | 显示「生成项目复盘报告」按钮 + 「历史报告」按钮 |
| 2.2 | 进入一个 **status!=completed** 的项目详情 | 不显示报告生成按钮 |
| 2.3 | 点击「生成项目复盘报告」 | 按钮 loading，请求发送 |
| 2.4 | 生成完成 | 弹窗展示报告内容（项目名、客户、周期、财务数据、分析段落） |
| 2.5 | **Provider=api**：分析段落为 AI 生成文本 | 有意义的分析内容 |
| 2.6 | **Provider=none**：分析段落显示占位文本 | 「（AI 分析不可用，请手动补充）」 |
| 2.7 | 点击「历史报告」 | 弹窗列表显示：生成时间、Provider、状态、版本号 |
| 2.8 | 再次点击「生成项目复盘报告」 | 新增版本（版本号 2），旧版本 is_latest=0 |
| 2.9 | 历史列表点击「查看」 | 展示该版本报告内容 |
| 2.10 | 历史列表点击「删除」 | 确认后删除，列表刷新 |

## 场景三：客户分析报告

| 步骤 | 操作 | 预期结果 |
|------|------|---------|
| 3.1 | 进入客户详情页 | 显示「生成客户分析报告」按钮 + 「历史报告」按钮 |

| 3.3 | 历史报告功能 | 同场景二的 2.7~2.10 |

## 场景四：交付包质检

| 步骤 | 操作 | 预期结果 |
|------|------|---------|
| 4.1 | 进入项目详情 -> 交付包 Tab | 每行交付包操作列新增「质检」按钮 |
| 4.2 | 点击某个交付包的「质检」按钮 | 按钮 loading，请求发送 |
| 4.3 | 质检完成（有问题） | 弹窗展示建议列表：优先级标签、问题标题、描述 |
| 4.4 | 质检完成（无问题） | 弹窗显示「质检通过，交付包完整」 |
| 4.5 | 测试缺失模型版本的交付包 | 建议列表包含「交付包缺少模型版本」(high) |
| 4.6 | 测试未绑定项目的交付包 | 建议列表包含「交付包未绑定项目」(high) |

## 场景六：边界与降级

| 步骤 | 操作 | 预期结果 |
|------|------|---------|
| 6.1 | **Provider=none** 时直接 POST `/api/v1/qa/ask` | 返回 403 |
| 6.2 | **Provider=api** 但 API Key 无效时发送问答 | 返回 503 或显示请求失败提示 |
| 6.3 | 生成报告时 LLM 填充失败 | 报告仍生成成功，分析段落为占位文本 |
| 6.4 | 对不存在的 package_id 运行质检 | 返回 404 |
| 6.5 | 对不支持的 report_type 生成报告 | 返回 400 |

---

## 测试顺序建议

1. **先启动服务** -> 确认基础功能正常
2. **场景五**（Agent 设置页）-> 确认 Provider 状态显示正确
3. **场景一**（自由问答）-> 需要 API Key
4. **场景二/三**（报告生成）-> 可用 none 和 api 两种模式分别测试
5. **场景四**（交付质检）-> 纯规则引擎，不依赖 LLM
6. **场景六**（边界降级）-> 最后验证异常情况
