## 零、关键语义边界定义（实施前必读，禁止跳过）

### 0.1 定位

模板 = Jinja2文本（存templates表）/ 数据 = 系统字段拼装（代码定义）/ 生成 = render() / 存储 = 快照

### 0.2 生成内容存储位置（禁止新建文档表）

quotations 和 contracts 表各新增三字段：generated_content / template_id / content_generated_at

### 0.3 内容快照与手工编辑规则（精确定义）

```python
# 1. generated_content 为快照，结构化字段变更不自动回写
# 2. 用户可通过 PUT /generated-content 手工编辑
# 3. 手工编辑后 content_generated_at 不变（保留最后生成时间）
# 4. 重新生成规则：
#    - generated_content 为空：直接生成
#    - generated_content 非空 + force=false（默认）：返回 HTTP 409（CONTENT_ALREADY_EXISTS）
#    - generated_content 非空 + force=true：覆盖旧快照，更新 content_generated_at
# 5. 内容冻结：
#    - quotations.status = accepted：不允许重新生成，即使 force=true 也不行
#    - contracts.status = active：同上
#    - 冻结状态下 GET /preview 仍然允许（不写库）
#    - 冻结状态下手工编辑也不允许
```

### 0.4 模板变量完整定义（必须与模板内容完全对齐）

### 0.5 合同草稿中 deliverables 的来源（解决与 v1.11 冲突）

```python
# deliverables_desc 在合同模板中为可选变量，默认空字符串
# 系统不自动从 deliverables 表提取内容
# 模板显示占位符"（请补充交付内容说明）"
# 用户手动填写后传入，v1.11 的 deliverables 表只用于交付包附件
```

### 0.6 草稿去重规则

```python
# 从项目生成报价草稿：
# 一个项目同一时间只允许一份 status=draft 且 template_id IS NOT NULL 的报价单
# 已存在时返回 HTTP 409（DRAFT_ALREADY_EXISTS），响应体包含已有 quotation_id

# 从报价生成合同：复用 v1.6 QUOTE_ALREADY_CONVERTED，一张报价只能转一次
```

### 0.7 模板删除规则

```python
# 默认模板不允许删除（TEMPLATE_IS_DEFAULT）
# 非默认模板可删除，历史 generated_content 快照不受影响（template_id 置 NULL，内容保留）
```

### 0.8 错误码边界（精确，两者互斥）

```python
# TEMPLATE_MISSING_REQUIRED_VARS：生成前校验，业务数据不完整（模板没问题，数据有问题）
# TEMPLATE_RENDER_FAILED：render() 执行时抛异常（数据没问题，模板有问题）
# 先校验数据再渲染，不会同时触发
```
