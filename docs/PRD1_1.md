\## 三、详细功能说明



\### 模块一：提醒管理中心



\*\*功能描述\*\*：统一管理所有时间敏感的提醒事项。区分"关键提醒"（年报/税期）和"普通提醒"，对关键提醒施加强制性约束，确保不被忽略。



\*\*用户故事\*\*：

\- 作为用户，我希望在一个地方看到所有待处理的提醒事项

\- 作为用户，我希望年报和税期提醒无法被忽视，逾期后会持续置顶

\- 作为用户，我希望合同到期和任务截止的提醒自动生成，无需手动添加

\- 作为用户，我希望处理完一条提醒后可以标记为已完成



\*\*功能清单\*\*：



| 功能 | 详细说明 | 验收标准 |

|------|----------|----------|

| 提醒列表 | 展示所有提醒，支持按类型和状态筛选 | 默认展示未完成提醒，逾期提醒置顶，其余按到期时间升序 |

| 新增手动提醒 | 用户手动创建自定义提醒 | 填写标题、类型、提醒日期即可创建 |

| 编辑提醒 | 修改提醒内容和日期 | 已完成的提醒不可编辑；关键提醒不可删除 |

| 标记完成 | 将提醒标记为已处理 | 标记后移入"已完成"分组，保留记录 |

| 删除提醒 | 仅限自定义提醒和非关键系统提醒 | 年报/税期类关键提醒无删除按钮 |

| 逾期升级 | 到期日过后仍未完成的提醒自动变更状态 | 状态变为"已逾期"，仪表盘置顶显示，标红高亮 |

| 仪表盘聚合 | 在仪表盘待办区展示近期未完成提醒 | 逾期提醒优先置顶，其次展示 7 天内到期提醒，最多 10 条 |

| 提醒配置 | 设置页配置年报和税期提醒的周期和截止日 | 配置保存后下一周期自动生效 |



\*\*提醒类型定义\*\*：



| 类型 | 是否关键提醒 | 生成方式 | 触发规则 | 默认配置 |

|------|------------|----------|----------|----------|

| 年报 / 审计提醒 | ✅ 是 | 系统周期生成 | 每年固定日期 | 每年 3 月 31 日 |

| 税期申报提醒 | ✅ 是 | 系统周期生成 | 按申报周期 | 月度，每月 15 日 |

| 合同到期提醒 | ❌ 否 | 系统自动生成 | 到期前 7 天 | 不可配置 |

| 任务截止提醒 | ❌ 否 | 系统自动生成 | 截止前 3 天 | 不可配置 |

| 文件到期提醒 | ❌ 否 | 系统自动生成 | 到期前 30 天 | 不可配置 |

| 自定义提醒 | ❌ 否 | 手动创建 | 用户指定日期 | 自由配置 |



\*\*数据字段\*\*：



| 字段 | 类型 | 必填 | 说明 |

|------|------|------|------|

| 提醒标题 | 文本 | ✓ | 最多 200 字符 |

| 提醒类型 | 枚举 | ✓ | 年报审计 / 税期申报 / 合同到期 / 任务截止 / 文件到期 / 自定义 |

| 是否关键提醒 | 布尔 | ✓ | 系统自动判断，年报/税期为 true |

| 提醒日期 | 日期 | ✓ | 到期或需处理的日期 |

| 状态 | 枚举 | ✓ | 待处理 / 已逾期 / 已完成 |

| 关联对象类型 | 枚举 | | 合同 / 任务 / 文件索引 / 无 |

| 关联对象 ID | 外键 | | 对应关联对象的 ID |

| 备注 | 长文本 | | 最多 1000 字符 |

| 完成时间 | 时间戳 | | 标记完成时自动记录 |

| 生成方式 | 枚举 | | 手动 / 系统自动 |



\*\*业务规则\*\*：

\- R-001（关键提醒强制性）：年报/税期类提醒不允许删除，未完成时持续显示在仪表盘，无法通过任何操作将其隐藏

\- R-002（逾期升级）：每日凌晨系统检查所有"待处理"提醒，过期未完成者状态自动变更为"已逾期"，仪表盘置顶标红

\- R-003（幂等保护）：同一合同/任务/文件在同一周期内不重复生成提醒

\- R-004（级联删除）：关联对象被删除时，对应系统自动提醒同步删除；关键提醒不受此规则影响

\### 模块二：文件资料索引



\*\*功能描述\*\*：管理公司重要文件的结构化元数据。不存储文件本身，只记录文件信息、版本、存放位置及关联业务对象。支持版本管理，同一文件多个版本（如历年审计报告）可关联归组。



\*\*用户故事\*\*：

\- 作为用户，我希望把公司所有重要文件的信息录入系统，方便检索

\- 作为用户，我希望知道每份文件的版本及当前有效版本是哪个

\- 作为用户，我希望文件可以关联到具体的合同、项目或财务记录

\- 作为用户，我希望证照快要到期时能收到提醒



\*\*功能清单\*\*：



| 功能 | 详细说明 | 验收标准 |

|------|----------|----------|

| 新增文件索引 | 录入文件元数据 | 必填项校验，文件名称不可为空 |

| 文件列表 | 展示所有文件索引，支持筛选和搜索 | 按类型筛选，按文件名搜索，同组版本可展开收起 |

| 编辑文件索引 | 修改文件元数据 | 修改成功后列表即时更新 |

| 删除文件索引 | 删除记录 | 二次确认，同步删除关联提醒 |

| 新增版本 | 为已有文件新增新版本记录 | 新版本创建后自动标记为当前有效版本 |

| 到期高亮 | 即将到期的文件高亮显示 | 到期前 30 天标记"即将到期"，过期标红 |



\*\*数据字段\*\*：



| 字段 | 类型 | 必填 | 说明 |

|------|------|------|------|

| 文件名称 | 文本 | ✓ | 最多 200 字符，如"营业执照正本" |

| 文件类型 | 枚举 | ✓ | 见下方枚举定义 |

| 版本号 | 文本 | | 最多 50 字符，如"v1"、"2026版"、"第二次修订" |

| 是否当前有效版本 | 布尔 | ✓ | 同一文件组内仅一个版本为有效版本 |

| 签署 / 签发日期 | 日期 | | |

| 到期日期 | 日期 | | 填写后自动触发到期提醒 |

| 实际存放位置 | 文本 | | 最多 500 字符，如"D:/公司文件/证照/（已同步至NAS）" |

| 关联对象类型 | 枚举 | | 合同 / 项目 / 财务记录 / 无 |

| 关联对象 ID | 外键 | | 对应关联对象的 ID |

| 经办机构 | 文本 | | 最多 200 字符 |

| 备注 | 长文本 | | 最多 1000 字符 |



\*\*文件类型枚举\*\*：

class FileType(str, Enum):

&#x20;   ANNUAL\_REPORT = "annual\_report"          # 年度报告

&#x20;   OTHER = "other"                          # 其他

```



\*\*业务规则\*\*：

\- 同一文件可存在多个版本记录，通过"文件名称 + 文件类型"分组，同组内仅一个版本标记为当前有效版本

\- 新增版本时，系统自动将该组内原有效版本标记为历史版本

\- 填写了到期日期的文件，到期前 30 天自动在提醒管理中心生成"文件到期提醒"（幂等保护）

\- 删除文件索引记录时，对应的自动提醒同步删除



\*\*检查点\*\*：

\- ✅ 版本分组逻辑是否正确（同名+同类型为一组）？

\- ✅ 新增版本后原有效版本是否自动降级为历史？

\- ✅ 到期提醒联动是否正确且有幂等保护？

\- ✅ 删除时关联提醒是否同步清理？

\### 模块三：财务记录扩展（v1.0 模块升级）



\*\*功能描述\*\*：在 v1.0 财务收支记录基础上，新增资金来源、业务说明、关联记录三个字段。关联记录用于实现垫付/借款的流转闭环，确保每笔非对公账户的资金都有来龙去脉可查。



\*\*原有规则（FIN-001 至 FIN-005）保持不变，新增以下内容：\*\*



\*\*新增数据字段\*\*：



| 字段 | 类型 | 必填规则 | 说明 |

|------|------|----------|------|

| 资金来源 | 枚举 | 支出必填，收入选填 | 见下方枚举定义 |

| 业务说明 | 文本 | 资金来源为个人垫付/借款时必填 | 最多 500 字符，说明垫付或借款原因 |

| 关联记录 ID | 外键 | 选填 | 关联同一财务记录表中的另一条记录 |

| 关联说明 | 文本 | 填写关联记录时必填 | 最多 200 字符，说明关联关系，如"报销张三2026-03-01垫付" |



\*\*资金来源枚举\*\*：



```python

class FundingSource(str, Enum):

&#x20;   COMPANY\_ACCOUNT = "company\_account"      # 对公账户

&#x20;   PERSONAL\_ADVANCE = "personal\_advance"    # 个人垫付

&#x20;   REIMBURSEMENT = "reimbursement"          # 报销

&#x20;   LOAN = "loan"                            # 借款

&#x20;   LOAN\_REPAYMENT = "loan\_repayment"        # 借款归还

&#x20;   OTHER = "other"                          # 其他

```



\*\*关联记录使用场景说明\*\*：



| 场景 | 操作方式 |

|------|----------|

| 个人垫付后报销 | 报销记录（收入）→ 关联原垫付记录（支出），关联说明填写"报销2026-03-01垫付款" |

| 借款后归还 | 归还记录（支出，资金来源=借款归还）→ 关联原借款记录（收入），关联说明填写"归还2026-01-15借款" |

| 对公转个人备用金 | 备用金支出记录 → 后续报销时关联此记录 |



\*\*新增业务规则\*\*：

\- FIN-006：支出记录的资金来源为必填项，未填时提交被拦截

\- FIN-007：资金来源为"个人垫付"或"借款"时，业务说明为必填项

\- FIN-008：填写关联记录 ID 时，关联说明为必填项

\- FIN-009：关联记录只能关联同一数据库中已存在的财务记录，不允许自关联

\- FIN-010：月度报表新增资金来源维度汇总统计（各来源支出合计），并展示未闭环的垫付/借款记录数量提示



\*\*数据库迁移要求\*\*：

\- v1.0 历史财务记录中，资金来源字段默认填充为"对公账户"，其余新增字段默认为空

\- 迁移脚本须在不丢失任何历史数据的前提下执行，迁移后需验证记录总数一致



\*\*检查点\*\*：

\- ✅ 支出记录资金来源是否强制校验？

\- ✅ 垫付/借款时业务说明是否强制校验？

\- ✅ 关联记录是否禁止自关联？

\- ✅ 月度报表是否新增资金来源维度和未闭环提示？

\- ✅ 数据库迁移后历史记录是否完整且总数一致？

\### 模块四：系统默认数据模板



\*\*功能描述\*\*：系统首次初始化（或重置）时，自动预填默认配置和文件清单，降低用户冷启动成本，避免空系统无从下手。



\*\*默认提醒配置（写入 reminder\_settings 表）\*\*：



| 提醒类型 | 默认配置 |

|----------|----------|

| 年报/审计提醒 | 每年 3 月 31 日触发 |

| 税期申报提醒 | 月度申报，每月 15 日触发（适用小规模纳税人） |



\*\*默认文件索引清单（写入 file\_indexes 表，所有字段除文件名和类型外留空）\*\*：



| 文件名称 | 文件类型 | 备注提示 |

|----------|----------|----------|

| 营业执照正本 | 营业执照 | 请填写有效期和存放位置 |

| 公司章程 | 公司章程 | 请填写签署日期和存放位置 |

| 数标园入驻协议 | 租赁/入驻协议 | 请填写到期日期，到期前 30 天系统将自动提醒 |

| 税务备案回执 | 税务备案回执 | 请填写经办机构和存放位置 |

| 首次审计报告 | 审计报告 | 请填写签发日期和存放位置 |



\*\*业务规则\*\*：

\- 默认数据模板仅在系统初次启动（数据库为空）时自动写入，不覆盖已有数据

\- 用户可自由编辑或删除默认预填的文件索引记录

\- 默认提醒配置可在设置页修改



\*\*检查点\*\*：

\- ✅ 系统初次启动后默认文件清单是否自动出现？

\- ✅ 默认提醒配置是否正确写入 reminder\_settings 表？

\- ✅ 重复启动时是否不覆盖已有数据？
