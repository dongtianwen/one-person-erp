# 技术设计决策

## 1. 技术栈选择

### 后端: FastAPI
- **理由**: 异步高性能，自动 Swagger 文档，类型安全
- **替代方案**: Flask (同步性能差), Django (过重)

### ORM: SQLAlchemy 2.0
- **理由**: 现代异步支持，类型安全，活跃维护
- **替代方案**: Tortoise ORM (生态小), SQLModel (不够灵活)

### 数据库: SQLite → PostgreSQL
- **理由**: 起步零配置，一人公司数据量小，后续无缝迁移
- **迁移策略**: 使用 SQLAlchemy 抽象层，切换数据库仅改连接字符串

### 前端: Vue 3 + Element Plus
- **理由**: 组合式 API 灵活，Element Plus 企业级组件开箱即用
- **替代方案**: React + Ant Design (学习曲线陡), Svelte (生态小)

## 2. 架构决策

### 分层架构
```
API 路由 → 业务逻辑 → CRUD 操作 → 数据库
```
- **理由**: 职责清晰，易于测试，便于扩展

### JWT 认证
- **理由**: 无状态，适合前后端分离
- **Token 策略**: Access Token (30分钟) + Refresh Token (7天)

### 软删除
- **理由**: 数据可恢复，审计追踪
- **实现**: 所有模型添加 `is_deleted` 字段

## 3. 数据库设计决策

### 外键约束
- **启用**: SQLite 默认不启用，需手动开启 `PRAGMA foreign_keys = ON`
- **理由**: 数据完整性

### 时间戳
- **策略**: 所有模型包含 `created_at` 和 `updated_at`
- **实现**: 基类模型自动处理

## 4. 安全决策

### 密码加密
- **算法**: bcrypt
- **理由**: 行业标准，抗暴力破解

### SQL 注入防护
- **策略**: ORM 参数化查询，禁止原生 SQL

### XSS 防护
- **策略**: 前端输入转义，Vue 自动处理
